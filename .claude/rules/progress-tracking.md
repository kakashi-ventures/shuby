# Progress Tracking

## After every commit that completes items from `docs/REMAINING-WORK.md`:
1. Mark completed items as `[x]` in `docs/REMAINING-WORK.md`
2. Add new items discovered during implementation
3. Commit the doc update (same commit or immediately after)
4. Push if pushing other work

## Do NOT batch progress updates — update immediately after each completion.

## Source of truth hierarchy:
- `docs/REMAINING-WORK.md` — prioritized task list (maintained by devs)
- `docs/AUDIT-REPORT.md` — full gap analysis (regenerated by `/shuby-audit`)
- `.claude/HANDOFF.md` — session context (updated by `/shuby-handoff`)
